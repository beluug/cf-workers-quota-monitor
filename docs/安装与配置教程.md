# 零基础安装与配置

不需要开发经验，照着下面做即可。整个过程大约 3 分钟。

> 安全原则：不要把 Cloudflare 密码、Global API Key 或 API Token 发给任何人，也不要上传到 GitHub。

## 1. 安装 App

1. 从 GitHub Releases 下载 `CF-Quota-Monitor-v1.3.1.apk`。
2. 在手机文件管理器中点击 APK。
3. 如果系统询问，只为当前文件管理器临时允许“安装未知应用”。
4. 安装完成后打开 App。应用锁默认关闭，可在“设置”中自行开启。

## 2. 找到 Account ID

1. 浏览器打开 [Cloudflare Dashboard](https://dash.cloudflare.com) 并登录。
2. 选择要监控的账号。
3. 打开 **Workers & Pages**。
4. 找到并复制 **Account ID**，它通常是 32 位字母和数字。

Account ID 不是邮箱、域名、Zone ID 或 Worker 名称。

## 3. 创建只读 API Token

1. 打开 Cloudflare **个人资料 → API Tokens**。
2. 点击 **Create Token → Create Custom Token**。
3. Token 名称填写 `CF额度监控`。
4. 只添加下面一项权限：

```text
Account → Account Analytics → Read
```

5. 账号资源只选择需要监控的账号。
6. 创建并复制 Token。

不要添加 Edit、DNS 或 Workers Scripts 权限，也不要使用 Global API Key。

## 4. 添加账号

1. 打开“CF额度监控”。
2. 点击右下角橙色 **＋**。
3. 填写一个容易识别的备注，例如“主账号”。
4. 粘贴 Account ID 和只读 API Token。
5. 每日额度保持 `100000`。
6. 点击 **保存并查询**。

多个账号重复以上步骤即可。建议每个账号分别创建一个只限制该账号的 Token。

## 5. 在Android和Windows之间迁移账户

1. 打开 **设置 → 账户迁移 → 导出账户**。
2. 勾选要导出的账户，设置至少8位的备份密码，然后保存`.cfqm`文件。
3. 把`.cfqm`文件复制到另一台设备，在应用中打开 **设置 → 账户迁移 → 导入账户**。
4. 选择文件并输入同一个备份密码，确认账户预览和重复账户处理方式后导入。

`.cfqm`里的Account ID和API Token已加密，但仍应像密码文件一样妥善保管。备份密码不会上传或保存，忘记后无法找回。

如果旧版Android已经生成了`.cfqm.json`文件，Windows v1.0.2及以上可以直接选择并导入，不需要手动改名。

## 常见问题

### Token 无效或权限不足

确认 Token 权限为 `Account Analytics → Read`，资源范围包含正确账号。在 App 中点击“编辑”即可替换 Token。

### 数据显示为 0 或比控制台少

Cloudflare Analytics 可能延迟数分钟。点击右上角 **↻** 重新查询。App 展示的是近实时估算，不要把最后少量额度安排得过满。

### 如何设置后台更新？

打开 App 右上角“设置”，开启“允许后台刷新”，再选择 15 分钟至 24 小时的刷新频率。Android 可能因省电模式或无网络略微延迟任务；不需要后台刷新时可随时关闭。

### 如何切换语言和应用锁？

打开右上角“设置”。语言首次自动跟随系统，也可手动选择中文、英语、俄语、意大利语、法语、西班牙语或阿拉伯语。开启“应用锁”后，每次离开再进入都需要指纹、面容或系统锁屏密码。

### 如何彻底删除凭证？

删除 App 中的账号，或直接卸载 App。对应的本机加密数据会一并删除。
