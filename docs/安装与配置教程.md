# 零基础安装与配置

不需要开发经验，照着下面做即可。整个过程大约 3 分钟。

> 安全原则：不要把 Cloudflare 密码、Global API Key 或 API Token 发给任何人，也不要上传到 GitHub。

## 1. 安装 App

1. 从 GitHub Releases 下载 `CF额度监控-v1.0.0.apk`。
2. 在手机文件管理器中点击 APK。
3. 如果系统询问，只为当前文件管理器临时允许“安装未知应用”。
4. 安装完成后打开 App，用指纹、面容或锁屏密码进入。

如果手机没有锁屏密码，请先在手机设置中添加 PIN、图案、指纹或面容。

## 2. 找到 Account ID

1. 浏览器打开 [Cloudflare Dashboard](https://dash.cloudflare.com) 并登录。
2. 选择要监控的账号。
3. 打开 **Workers & Pages**。
4. 找到并复制 **Account ID**，它通常是 32 位字母和数字。

Account ID 不是邮箱、域名、Zone ID 或 Worker 名称。

## 3. 创建只读 API Token

1. 打开 Cloudflare **个人资料 → API Tokens**。
2. 点击 **Create Token → Create Custom Token**。
3. Token 名称填写 `CF额度监控`。
4. 只添加下面一项权限：

```text
Account → Account Analytics → Read
```

5. 账号资源只选择需要监控的账号。
6. 创建并复制 Token。

不要添加 Edit、DNS 或 Workers Scripts 权限，也不要使用 Global API Key。

## 4. 添加账号

1. 打开“CF额度监控”。
2. 点击右下角橙色 **＋**。
3. 填写一个容易识别的备注，例如“主账号”。
4. 粘贴 Account ID 和只读 API Token。
5. 每日额度保持 `100000`。
6. 点击 **保存并查询**。

多个账号重复以上步骤即可。建议每个账号分别创建一个只限制该账号的 Token。

## 常见问题

### Token 无效或权限不足

确认 Token 权限为 `Account Analytics → Read`，资源范围包含正确账号。在 App 中点击“编辑”即可替换 Token。

### 数据显示为 0 或比控制台少

Cloudflare Analytics 可能延迟数分钟。点击右上角 **↻** 重新查询。App 展示的是近实时估算，不要把最后少量额度安排得过满。

### 为什么退出后不更新？

这是设计行为。App 不注册后台任务，只在打开或手动刷新时联网，更省电也更保护隐私。

### 如何彻底删除凭证？

删除 App 中的账号，或直接卸载 App。对应的本机加密数据会一并删除。
